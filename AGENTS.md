# AGENTS.md

## Repo Working Rules

- Keep a short recovery log in `WORKLOG.md`.
- Append to `WORKLOG.md` at major checkpoints, before stopping work, and after any user-visible release or deployment step.
- Record at least:
  - current status
  - what changed
  - uncommitted files
  - next recommended step
- Do not assume session memory will persist; use `WORKLOG.md` as the source of truth for resuming work.
